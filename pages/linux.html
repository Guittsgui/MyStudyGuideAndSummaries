Linux

Para começar a falar de linux precisamos deixar claro que Linux é 100% open source, de forma que qualquer pessoa pode desenvolver, alterar, copiar uma versão do Linux. Essas versões do linux são chamada de Distribuições. Por isso é importante citar as mais famosas : Ubunto, Debian, CentOS, Fedora, Alpine, a alpine não é muito famosa, porém ela é de extrema relevância dentro do universo Docker, pois ela é extremamente leve.


Instalando o Linux

Dado nosso estudo em Docker, já sabemos que não precisamos ter exatamente o linux instalado dentro do nosso computador, e sim podemos criar um container, que dentro dele rode Linux em qualquer versao que no nosso caso instalaremos Ubuntu e em seguida iremos praticar os principais comandos de Linux. Basicamente acessaremos o DockerHub nele realizaremos o download/PULL da imagem ubunto para dentro de nossa máquina, e em seguida, criaremos um container com essa imagem e a rodaremos. Esse é o objetivo atual para que posasmos comecar a trabalhar c linux. Aqui podemos trabalhar e duas formas: baixar a imagem salvar em nossa máquina e rodar ela, ou então podemos somos rodar ela no sistema Interativo (IT). Na verdae tanto faz a forma que escolhermos, nesse caso iremos rodar ela sem baixar, já que nós iremos apenas praticar os conceitos. Para isso comecamos no nosso terminal: docker ps para listar os Containers rodando. Também temos o docker ps -a visualizaremos todos os containers que existem sem estar rodando. Se fossemos apenas baixar para nosas maquina, colocariamos o comando: docker pull ubuntu, porém como iremos utilizar o modo interativo, colocamos docker run -it ubuntu. Dessa forma, poderemos reparar que logo em seguida, o terminal irá mudar e automaticamente estaremos dentro do root de um container Linux. Aqui já poderemos comçar a anotar os comandos. Aqui basicamente rodamos e acessamos uma imagem do dockerhub dentro de nossa máquina.


Pacotes

Quando compramos um notebook novo por exemplo, ele provavelmente nao irá vir com os aplicativos que utilizamos no dia a dia, como o vsCode, Docker, etc etc. Dentro do linux, também temos que adicionar nossos programas, que aqui sao chamados de Packages. Para comecarmos precisamos ver quais pacotes que já vieram por padrao com nossa distribuicao linux, utilizando o comando: apt list. Sempre que pegamos uma versao linux do zero, a primeira coisa que precisamos fazer é atualizar ele. Para isso utilizamos o comando; apt update. Agora que ja entedemos o que são os packages, vamos supor que queiramos criar um arquivo de texto, como criaremos ? Para isso existe um package chamado nano, e ao digitarmos nano ele nao sera apresentado. Para instalarmos, usando o comando apt install nano. Agora para criarmos um arquivo de texto, podemos colocar o comando: nano teste.txt e assim o arquivo será criado.


Sistemas de Arquivos Linux.

Dentro do linux precisamos entender o sistema de arquivos e de diretórios de pasta. Ele é bem parecido com o do windows porém com algumas diferencas. No windows tudo comeca com c:/...... e tudo é criado a partir dai. No Linux tudo comeca a partir do Root. Depois da barra temos acesso a estrutura de diretórios como por exemplo: /bin, /dev, / etc . o Bin foi criado para armazenar comandos binários que precisam ser disponibilizado para todos os usuários, como o LS por exemplo, e assim por diante, cada um desses locais. Podemos então procurar no google esta pesquisa para nos aprofundarmos mais.


Navegando no Linux

Aqui iremos aprender a navegar pelo ambiente Linux. O primeiro comando é o LS que irá listar tudo o que há dentro do diretório atual.

Resumo Comandos:

  • WHOAMI - Mostrar qual usuário atual.
  • ECHO - Envia alguma coisa para a tela, Ex: Echo hi, retornará o Hi na tela.
  • ECHO $0 - Informará qual o diretório que voce está.
  • HISTORY - Infomará quais os últimos comandos acessaram.

    • Packages:

  • APT LIST - Mostrará a lista de tudo que está instalado.
  • APT UPDATE - Acessará o ubunto e atualizará os packages para os mais atualizados e usados.
  • APT INSTALL NOMEPACOTE(NANO) Instala qualquer pacote.
  • APT REMOVE NOMEPACOTE(NANO) Desinstala qualquer pacote.

    • Navegation:

  • LS Lista tudo que há dentro do diretório atual.
  • LS -A Lista tudo + os arquivos ocultos.
  • LS -1 Lista tudo um embaixo do outro
  • LS -l Passa as informacoes detalhadas do que há no diretório.
  • PWD Nos mostra o caminho que estamos para o diretório atual.
  • CD /dirname Serve para acessarmos um diretorio em questao no dirname. (CD = ChangeDirectory).
  • CD .. Volta para o diretório ROOT.
  • LS /direname Lista o que há dentro de um diretório sem navegar para ele.

    • CreateFiles&Dir:

  • CD ˜ Navega para o diretório Home dentro do Root.
  • MKDIR NOMEDODIRETORIO Criar um diretório .
  • MV NOMEDIR NOVONOMEDIR Renomeia o nome de um diretório para outro.
  • TOUCH NOME.TXT Cria um arquivo.
  • RM NOMEDOARQUIVO Remove um arquivo
  • RM -R NOMEDODIRETORIO Remove um diretório.
  • RM HI* irá remover tudo que comeca com hi.

    • CreatingFiles:

  • NANO NOMEARQUIVO.TXT Cria um arquivo txt.
  • CAT NOMEARQUIVO.TXT Visualiza o arquivo sem precisar abrir.

    • Redirecionamento no Linux:

  • CAT ARQUIVO.TXT > ARQUIVONOVO.TXT Copia tudo que tem em um arquivo e cria um novo com essas informacoes.

    • GREP

    (Procura por conteúdo dentro de Arquivos).
  • GREP HELLO NOMEARQUIVO.TXT Procura se dentro do arquivo tem "HELLO".
  • GREP HELLO ARQUIVO1.TXT ARQUIVO2.TXT Procura se no arquivo 1 e arquivo 2 possuem "HELLO".
  • GREP HELLO FILL* Procura se há Hello em todos os arquivos que comecem com FILL.
  • GREP -I -R HELLO Procura por diretórios.

    • FIND

    Procura por Arquivos.
  • FIND Irá mostrar tudo o que há no diretório incluindo arquivos ocultos.
  • FIND -TYPE F Mostra apenas os arquivos.
  • FIND -TYPE D Mostra apenas os diretórios.
  • FIND -TYPE F -NAME "DOCKER.TXT" Procura pelo arquivo em especficio.
  • FIND -TYPE F -NAME "ASN*" Procura por arquivos que comecem com ASN.

    • MultipleCommands

  • MKDIR TESTE1 ; CD TESTE1; ECHO OK Executa os 3 comandos seguidos em uma linha, separados por ;
  • MKDIR TESTE1 && CD TESTE1 && ECHO OK Funciona igual o acima, porém irá parar quando apresentar error. ;

    • ManagingProcesses

  • PS Nos mostra nossos processos e suas infos.
  • SLEEP 5 Nao responde a linha de comando por 5 segundos.
  • KILL IDASERPARADO Mata o processo através de seu ID.

    • MannageingUsers

  • WHOAMI Mostra o usuário atual.
  • USERADD -M nomeUsuario Cria um usuário.
  • DOCKER EXEC -IT -U USERNAME ID BASH Abre o container logando com o usuário desejado. link

    • MannagingGroups

  • CAT /ETC/GROUP Lista todos os grupos.
  • GROUPS USUARIO Mostra a quais grupos o usuario em questao participa.
  • GROUPADD NOMEDOGRUPO Cria um novo grupo.
  • usermod -G nomedoGrupo nomeUsuario Adiciona o usuario dentro do novo grupo.

    • PermissionFiles

    Para falarmos de permissoes em arquivos/diretorios precisamos entender que um usuário basicamente pode Read: Ler um arquivo, Write: Escrever um arquivo e eXecute: Executar um arquivo. Link
